# Xiaohongshu Publisher

Create publish-ready 小红书图文内容 and publish it with the logged-in browser session.

## Workflow

1. Understand the post goal: traffic,记录,种草,总结,热点解读.
2. Convert source material into:
   - title
   - body copy
   - hashtag set
   - image plan
3. Generate or organize image assets into a dedicated folder.
4. Prefer concise, platform-native Chinese copy.
5. Reuse the existing logged-in 小红书 page/session whenever possible. Do not repeatedly reopen login pages.
6. Before publishing, verify:
   - title filled
   - body filled
   - images uploaded
   - no text overflow in generated images
7. Publish or leave as draft depending on user request.

## Rules

- Default to 图文笔记 unless the user explicitly wants video.
- Prefer 3-7 images for information-heavy posts.
- Keep title sharp and readable.
- Keep body scannable: short paragraphs, no giant blocks.
- If publishing requires login, ask the user to complete login once, then continue by reusing that session.
- Avoid risky repeated browser relaunches.

## Common Output

### 1. Lifestyle / daily share
- warm title
- short first-person body
- natural hashtags

### 2. News / summary post
- headline title
- bullet-like body
- card/infographic image plan

### 3. Product / recommendation
- problem
- experience
- recommendation
- CTA
