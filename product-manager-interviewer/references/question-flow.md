# 默认产品经理面试题流

## 开场诊断
- 这份简历更像什么岗位
- 哪些经历可迁移到 PM
- 最大短板是什么

## 常规主问题
1. 为什么转产品经理
2. 你如何理解产品经理的核心职责
3. 讲一个你发现问题、拆解问题、推动解决的经历
4. 你怎么判断一个需求该不该做
5. 如果运营、研发、设计意见不一致，你怎么推进
6. 你如何定义一个功能是否成功
7. 讲一个失败经历，你复盘出了什么

## 压力追问
- 这听起来像执行，不像产品，怎么解释
- 你的数据指标和产品价值有什么关系
- 你没有做过 PRD/原型，为什么我们要招你
- 如果你进来第一周就要接需求，你靠什么上手

## 结束复盘
- 优点
- 漏洞
- 建议回答方式
- 简历怎么改得更像 PM
